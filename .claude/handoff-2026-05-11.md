## Important context
- Admin panel login is separate from Supabase. It uses an HMAC-signed cookie. The username, password and signing secret are env-only (`ADMIN_USERNAME`, `ADMIN_PASSWORD`, `ADMIN_SESSION_SECRET`).
- The CMS lives in tissu-agent, not in this repo. PUTs must be wrapped as `{ payload: {...} }` — already handled in `lib/admin-content.ts`.
- Inventory updates use `PUT /api/inventory/{id}` with **query params**, not a JSON body — already handled in `lib/admin-inventory.ts`.
- Stock decrement is tracked via the `_stockReserved` flag inside `Order.shippingAddress` JSON, to avoid a Prisma migration.
- Telegram notification is fire-and-forget (`(async () => {})()`). This is safe on Railway's long-running Node runtime; would need to switch to `waitUntil` if we ever moved to Vercel functions.
